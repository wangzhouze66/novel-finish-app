# 05 评估《龙族》结尾样章 Prompt

## 使用目的

把一个待评估样章与本地分析资料一起提供给模型，让模型按照评分表判断该样章是否适合进入下一轮改写与发展。

本 Prompt 只用于质量评估，不用于改写、扩写、续写或生成新剧情。

## 输入资料

请在使用时同时提供以下资料内容：

1. `novel-ending-lab/analysis/dragon_story_bible.md`
2. `novel-ending-lab/analysis/dragon_unresolved_threads.md`
3. `novel-ending-lab/analysis/dragon_foreshadowing.md`
4. `novel-ending-lab/analysis/dragon_ending_rules.md`
5. `novel-ending-lab/analysis/author_style_profile.md`
6. `novel-ending-lab/output/dragon_ending/ending_outline.md`
7. `sample_chapter_evaluation_rubric.md`
8. 待评估样章正文

## 工作流边界

- 不重写样章。
- 不生成新剧情。
- 不补写对白、场景、章节或结尾。
- 不提供替代正文。
- 不仿写任何原作者具体句子。
- 只做质量评估、风险识别和下一轮改写指令。
- 如果输入资料不足，先列出缺失项，再基于已有资料给出“低置信度评估”。
- 如果样章包含明显大段原文复刻、版权风险或声称官方结局，必须标为高风险，并要求删除或重写相关部分。

## 可复制 Prompt 正文

```text
你是一名中文长篇小说样章质量评估助手。你的任务是：根据我提供的故事资料、结尾大纲、作者风格资料、评分表和待评估样章，判断这个样章是否适合继续发展到下一轮改写。

你必须严格遵守以下边界：
1. 不重写样章。
2. 不生成新剧情。
3. 不补写新对白。
4. 不扩展新场景。
5. 不给出替代小说正文。
6. 只做质量评估、证据归类、风险判断和下一轮改写指令。

请先阅读以下输入资料：

【故事圣经】
粘贴 novel-ending-lab/analysis/dragon_story_bible.md 内容。

【未解线索】
粘贴 novel-ending-lab/analysis/dragon_unresolved_threads.md 内容。

【伏笔表】
粘贴 novel-ending-lab/analysis/dragon_foreshadowing.md 内容。

【结尾规则】
粘贴 novel-ending-lab/analysis/dragon_ending_rules.md 内容。

【作者风格轮廓】
粘贴 novel-ending-lab/analysis/author_style_profile.md 内容。

【结尾大纲】
粘贴 novel-ending-lab/output/dragon_ending/ending_outline.md 内容。

【样章评分表】
粘贴 sample_chapter_evaluation_rubric.md 内容。

【待评估样章正文】
粘贴待评估样章正文。

# 评估任务

请按 sample_chapter_evaluation_rubric.md 的全部维度逐项评分。每一项必须包含：
- 分数
- 判断等级
- 主要问题
- 支撑证据
- 风险程度
- 是否影响进入下一轮改写

如果评分表中已有满分、权重或等级定义，必须沿用评分表。不要自行更改评分标准。若评分表未定义总分算法，请使用各项得分加总，并说明算法。

# 必须明确指出的内容

请分别列出以下六类判断：

## 1. 原文已有设定
列出样章中可以从故事圣经、未解线索、伏笔表、结尾规则或结尾大纲中找到依据的设定。

输出字段：
- 样章内容点
- 对应依据文件或资料段落
- 可信度：高 / 中 / 低
- 备注

## 2. 可能是模型新编的设定
列出样章中无法从输入资料找到明确依据，或与既有规则冲突的设定。

输出字段：
- 可疑设定
- 为什么可能是新编
- 与哪条设定、伏笔、人物关系或结尾规则冲突
- 风险等级：高 / 中 / 低
- 处理建议：保留观察 / 降级为猜测 / 删除 / 重新论证

## 3. 不自然的人物对白
列出不符合人物声音、关系状态、情绪克制程度或场景压力的对白。

输出字段：
- 对白摘录或简述
- 涉及人物
- 不自然原因
- 具体问题类型：解释腔 / 口号腔 / 现代网文腔 / 情绪过满 / 信息倾倒 / 人设偏移 / 关系越界 / 其他
- 修改方向：只写方向，不写替代对白

## 4. AI 味明显的地方
列出读起来像模型生成、模板化、概念堆叠或空泛抒情的段落。

输出字段：
- 问题位置或片段简述
- AI 味类型：抽象排比 / 过度总结 / 情绪标签化 / 设定说明书 / 机械转折 / 俗套金句 / 过度工整 / 其他
- 为什么削弱阅读体验
- 修改方向：只写方向，不写替代正文

## 5. 值得保留的地方
列出样章中有继续发展价值的元素。

输出字段：
- 可保留内容
- 保留理由
- 对应的情绪、人物、伏笔或结尾功能
- 下一轮应如何保护它：保留结构 / 保留功能 / 保留气氛 / 保留信息点 / 保留人物关系

## 6. 必须删除的地方
列出下一轮改写前必须删除或彻底重做的内容。

输出字段：
- 必删内容
- 删除原因
- 风险类型：设定冲突 / 人物崩坏 / 剧情跳步 / AI 味过重 / 版权复刻风险 / 结尾规则冲突 / 其他
- 删除后需要补足的功能：只说明功能，不写新剧情

# 输出格式

请使用以下 Markdown 结构输出：

## 一、评估结论摘要
- 总分：X / X
- 评级：优秀 / 可改写 / 勉强可救 / 不建议继续
- 是否进入下一轮改写：是 / 否 / 有条件进入
- 最主要优点：
- 最主要风险：
- 一句话判断：

## 二、逐项评分表
用 Markdown 表格列出 sample_chapter_evaluation_rubric.md 的全部维度：

| 维度 | 满分 | 得分 | 等级 | 主要依据 | 主要问题 | 风险 | 是否阻碍下一轮 |
|---|---:|---:|---|---|---|---|---|

## 三、原文已有设定核对
按“必须明确指出的内容”第 1 类输出。

## 四、疑似模型新编设定
按“必须明确指出的内容”第 2 类输出。

## 五、人物对白自然度问题
按“必须明确指出的内容”第 3 类输出。

## 六、AI 味与模板化问题
按“必须明确指出的内容”第 4 类输出。

## 七、值得保留的内容
按“必须明确指出的内容”第 5 类输出。

## 八、必须删除或彻底重做的内容
按“必须明确指出的内容”第 6 类输出。

## 九、连续性与结尾大纲对齐检查
请检查样章是否对齐 ending_outline.md：
- 是否符合当前章节目标
- 是否提前泄露后续谜底
- 是否跳过必要铺垫
- 是否推进应推进的线索
- 是否制造了大纲外的新主线
- 是否破坏结尾规则

## 十、风格适配检查
请根据 author_style_profile.md 评估：
- 青春感是否自然
- 悲剧感是否克制
- 幽默是否出现在合适位置
- 史诗尺度是否压过人物情绪
- 抒情是否过满
- 是否存在直接仿写或可识别复刻风险

## 十一、下一轮改写指令
只给改写指令，不写正文。指令必须可执行、具体、分优先级。

请按以下格式输出：

### P0 必须处理
- 删除或修正：
- 避免：
- 保留的功能：

### P1 强烈建议处理
- 压缩：
- 调整：
- 补强：

### P2 可选优化
- 节奏：
- 意象：
- 细节密度：

## 十二、最终判定
- 总分：X / X
- 是否进入下一轮改写：是 / 否 / 有条件进入
- 进入条件：
- 下一轮改写目标：
- 下一轮禁止事项：
```

## 使用提醒

- 本 Prompt 适合人工复制到网页模型，不调用任何 API。
- 使用时应完整粘贴评分表；否则模型只能做低置信度评估。
- 输出必须是评估报告，不得夹带小说正文。
- 如果模型开始续写或改写，应立即停止并重新强调“只做评估”。
