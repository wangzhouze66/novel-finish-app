# 江南作品谱系分析 + 龙族结尾实验工程

> 当前阶段只建立本地研究流程、目录结构和 Markdown 模板；不生成正文、不仿写江南句子、不直接制作 App。

## 项目目标

本实验工程用于在本地分阶段完成以下工作：

1. 整理江南作品资料与《龙族》正典资料。
2. 提取江南创作 DNA：主题、人物模型、青春叙事、悲剧机制、战争与世界模型等。
3. 分析《龙族》现有剧情、人物、世界观、伏笔与未解线索。
4. 生成《龙族》结尾规则：什么必须回收、什么可以开放、什么不能违背。
5. 在规则完成后，再生成多个结局方案、大纲、章节结构；章节正文只在后续明确批准后进行。

## 核心原则

- **先资料，后分析，再规则，最后方案。**
- **不一键续写。** 所有生成都必须经过资料卡、风格谱系、故事圣经、未解线索、伏笔表和结尾规则。
- **不仿写句子。** 本工程只抽象分析叙事结构、主题机制、节奏功能、情感逻辑与人物模型。
- **多结局优先。** 先生成多个结尾路线并比较，再合成或选择最终路线。
- **模板先行。** 每个 Markdown 文件都是待填写容器，不应直接填入未经整理的长篇正文。

## 目录说明

| 目录 | 用途 | 当前阶段怎么用 |
| --- | --- | --- |
| `source/dragon_canon/` | 存放《龙族》各卷资料索引、摘要、人物与伏笔记录 | 只填结构化笔记，不粘贴大段原文 |
| `reference/jiangnan_corpus/` | 存放江南其他作品的作品卡与主题/人物/叙事机制分析 | 用于抽象创作 DNA，不用于句子级模仿 |
| `reference/literary_influences/` | 分析可能的文学影响源和与《龙族》的关系 | 记录“可借鉴的结构功能”，不是仿写风格 |
| `reference/reader_reception/` | 记录读者争议、期待、遗憾与接受史 | 区分事实、观点和个人偏好 |
| `analysis/` | 汇总作者谱系、《龙族》故事圣经、未解线索、伏笔与结尾规则 | 结尾方案生成前必须优先完成 |
| `output/dragon_ending/` | 存放结尾选项、总大纲与后续章节产物 | 当前只允许大纲，不生成正文 |
| `prompts/` | 存放可复用提示词模板 | 提示词必须要求输入资料与规则 |
| `scripts/` | 预留给后续本地整理脚本 | 当前不需要 App 代码 |

## 阶段门禁

### 阶段 1：资料索引

需要完成：

- 每卷《龙族》资料索引。
- 江南作品资料卡。
- 读者接受史与争议记录。
- 文学影响源的比较笔记。

未完成前，不进入结尾方案生成。

### 阶段 2：创作 DNA 分析

需要完成：

- 作者风格谱系。
- 青春诗学。
- 悲剧机制。
- 英雄模型。
- 爱情/关系模型。
- 战争与世界模型。

目标是提取“结构与功能”，不是复制表达。

### 阶段 3：《龙族》故事工程

需要完成：

- 故事圣经。
- 未解线索表。
- 伏笔表。
- 世界观规则与不可违背约束。

### 阶段 4：结尾规则

需要完成：

- 必须回收的线索。
- 可开放保留的问题。
- 不可违背的人物弧线。
- 世界观与主题约束。
- 风险清单。

### 阶段 5：多结局与大纲

只在阶段 1-4 完成后进行：

1. 生成多个结尾方案。
2. 比较方案优缺点。
3. 混合或选择方案。
4. 生成总大纲。
5. 再生成章节级大纲。
6. 最后才考虑章节正文。

## 写作与版权边界

本工程仅用于私人学习、文本结构研究、叙事机制分析和创作流程实验。

### 允许做的事

- 记录作品的主题、人物关系、叙事结构、情绪机制、世界观规则和伏笔类型。
- 生成作品分析卡、人物机制表、伏笔表、结尾规则表和高层大纲。
- 基于分析结果总结“少年感、悲剧机制、英雄模型、爱情结构、世界观压迫方式”等抽象创作规律。
- 使用用户自己拥有版权、已获授权或公版文本进行完整续写实验。
- 对受版权保护作品，只保留必要的短摘录、摘要和结构化分析笔记。

### 不允许做的事

- 不在本仓库保存、复制、上传或分发受版权保护作品的完整正文。
- 不输出大段可替代原著阅读的原文内容。
- 不宣称生成内容是原作者本人创作、官方续作或正版结局。
- 不以受版权保护作品的角色、世界观和正文生成内容进行公开发布、售卖或商业推广。
- 不直接仿写某位在世作者的句子、口癖、段落风格或可识别表达。
- 不把其他作家的作品混合成可替代原著的新文本。

### 对《龙族》相关资料的处理原则

- 《龙族》相关内容只作为私人研究样本，用于结构分析、伏笔整理、人物关系梳理和结尾规则设计。
- 本仓库只保存分析结果，不保存《龙族》全文。
- 若后续生成章节正文，只能用于私人草稿和内部实验，不公开传播，不商业化，不声明为官方结局。
- 公开展示时只能展示原创设定、抽象方法、流程模板和非侵权示例。

### 原始文本存放原则

受版权保护的原始小说文件应存放在本地私有目录中，不提交到 GitHub。

建议本地目录：

```text
local_raw_corpus/
├── dragon/
├── jiangnan/
└── literary_references/
```

该目录必须加入 .gitignore，避免误传。
