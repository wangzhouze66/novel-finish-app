# 手动模型投喂 Prompt：样章生成（DeepSeek 网页版复制版）

> 用途：把本仓库已有的《龙族》结尾分析资料手动整理为一个可复制到 DeepSeek 网页版的样章生成 prompt。  
> 边界：本文件不调用任何 API，不读取 API key，不生成真正样章；只提供人工复制使用的 prompt 模板。

## 使用方式

1. 打开以下输入资料，人工复制其中与目标章节相关的要点，不要复制大段原文。
2. 将本文件下方“可复制 Prompt”完整复制到 DeepSeek 网页版。
3. 将每个【请粘贴/概括】占位符替换为对应资料中的摘要、要点或章节目标。
4. 让模型只生成 1500-2500 中文字的一节样章，并附带自查，不要要求它输出完整结局或整章之后的内容。

## 输入资料清单

请在人工投喂前检查并摘取以下文件的信息：

- `novel-ending-lab/analysis/dragon_story_bible.md`
- `novel-ending-lab/analysis/dragon_unresolved_threads.md`
- `novel-ending-lab/analysis/dragon_foreshadowing.md`
- `novel-ending-lab/analysis/dragon_ending_rules.md`
- `novel-ending-lab/analysis/author_style_profile.md`
- `novel-ending-lab/output/dragon_ending/ending_outline.md`

## 可复制 Prompt

```markdown
你是一名中文长篇小说样章生成助手。请根据我手动提供的分析资料，生成一节 1500-2500 中文字的样章。

## 一、硬性边界

- 不调用 API。
- 不读取、询问或暗示需要 API key。
- 不照抄原作句子。
- 不输出大段原文。
- 不生成完整结局、完整最终卷或后续连续章节。
- 本次只生成“一节样章”，用于检验人物、情绪、伏笔推进和文风适配。
- 如果资料不足，请先在“风险点”里说明，不要自行把不确定内容当成既定正史。

## 二、样章目标

- 篇幅：1500-2500 中文字。
- 语言：中文。
- 目标章节/片段：
  【请粘贴或概括 `output/dragon_ending/ending_outline.md` 中要试写的章节编号、章节标题、本章目标、主要冲突、出场人物、必须推进的剧情线、必须回收或埋下的伏笔、情绪基调、结尾钩子、风险提示。】

## 三、故事连续性资料

### 1. 故事圣经摘要
【请粘贴或概括 `analysis/dragon_story_bible.md` 中与本节相关的世界观规则、人物现状、阵营关系、不可违背的连续性约束。避免粘贴原文大段内容。】

### 2. 未解线索摘要
【请粘贴或概括 `analysis/dragon_unresolved_threads.md` 中本节需要推进、暂缓或明确保留开放的线索。】

### 3. 伏笔资料摘要
【请粘贴或概括 `analysis/dragon_foreshadowing.md` 中本节需要回收、变奏或再次埋下的伏笔。】

### 4. 结尾规则摘要
【请粘贴或概括 `analysis/dragon_ending_rules.md` 中本节必须遵守的结尾方向、代价规则、情绪边界和禁区。】

## 四、作者风格与文体要求

请参考以下人工整理的风格资料，但不要模仿到逐句复刻，也不要照搬原文表达：

【请粘贴或概括 `analysis/author_style_profile.md` 中与本节相关的声音、节奏、意象、幽默位置、情绪克制方式、青春感、悲剧密度、史诗尺度、孤独/命运/失去时间等主题要求。】

写作时请特别注意：

- 先保留一点少年感、日常感或轻微幽默，再让情绪慢慢沉下去。
- 情绪表达要克制，不要把人物内心直接解释成口号。
- 场景要服务章节功能，不要只写气氛。
- 人物必须像在终局压力下行动的人，而不是站着讲设定的人。
- 伏笔推进要自然嵌入动作、物件、对话或场景变化中。
- 不要把“圆满”写成无代价爽文；如果本节接近圆满，也必须保留缺口、损耗或不安。

## 五、本节输出格式

请严格按以下结构输出：

# 样章标题

【生成一个适合本节的标题。】

# 样章正文

【生成 1500-2500 中文字正文。】

# 本节功能说明

- 本节在结尾大纲中的功能：
- 本节推进的主要冲突：
- 本节留下的结尾钩子或情绪余波：

# 人物一致性自查

- 路明非：
- 路鸣泽（如出场）：
- 楚子航（如出场）：
- 诺诺（如出场）：
- 恺撒（如出场）：
- 其他人物：

# 伏笔推进自查

- 本节回收/变奏的伏笔：
- 本节新埋或继续保留的伏笔：
- 暂时不处理但需要后续处理的线索：

# 风险点

- 可能偏离故事连续性的地方：
- 可能过度解释或信息密度过高的地方：
- 可能过度煽情、过度圆满或削弱悲剧密度的地方：
- 需要人工复核的设定或人物关系：
```

## 人工复核建议

- 先检查“样章正文”是否真的只是一节样章，而不是跳到完整结局。
- 再检查“人物一致性自查”是否能对应输入资料，而不是泛泛而谈。
- 最后检查“伏笔推进自查”和“风险点”，确认模型没有把资料不足的位置写成确定结论。
